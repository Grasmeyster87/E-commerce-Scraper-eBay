Для проєкту такого рівня, який ви плануєте виставити в портфоліо на Upwork, важливо уникати «спагеті-коду». Оскільки ви прихильник модульності, структура має чітко розділяти керування браузером, логіку парсингу та обробку даних.

Ось професійна структура для **eBay Scraper** на Node.js + Puppeteer:

### Архітектура проєкту

```text
ebay-scraper-nodejs/
├── src/
│   ├── config/
│   │   └── selectors.js       # CSS/XPath селектори (щоб не хардкодити в логіці)
│   ├── services/
│   │   ├── browser.js         # Ініціалізація та налаштування Puppeteer
│   │   ├── scraper.js         # Основний клас Scraper (навігація, пагінація)
│   │   └── parser.js          # Логіка вилучення даних з конкретних елементів
│   ├── utils/
│   │   ├── logger.js          # Красиве логування в консоль
│   │   └── fileHandler.js     # Збереження в JSON/CSV або SQLite
│   └── index.js               # Точка входу (App Entry Point)
├── data/                      # Папка для згенерованих файлів (CSV/JSON)
├── .env                       # Константи (URL, затримки, шлях до браузера)
├── .gitignore
├── package.json
└── README.md
```

### Деталізація модулів

#### 1. `src/config/selectors.js`
eBay часто змінює класи. Винесення їх в окремий конфіг дозволить оновити скрапер за 10 секунд без переписування логіки.
```javascript
export const selectors = {
    productCard: '.s-item__wrapper',
    title: '.s-item__title',
    price: '.s-item__price',
    link: '.s-item__link',
    paginationNext: '.pagination__next'
};
```

#### 2. `src/services/browser.js`
Відповідає за створення екземпляра браузера. Тут варто підключити `puppeteer-extra-plugin-stealth`, щоб eBay не блокував вас як бота.
*   **Функції:** `initBrowser()`, `createNewPage()`.

#### 3. `src/services/parser.js`
Цей модуль отримує "сирий" елемент або сторінку і повертає чистий об'єкт з даними.
*   **Метод:** `parseProduct(element)` — повертає `{ name, price, url }`.

#### 4. `src/services/scraper.js` (Клас `EbayScraper`)
Основний «мозок» програми.
*   **Методи:** 
    *   `search(query)` — вводить запит у рядок пошуку.
    *   `scrapePage()` — циклом проходить по товарах на сторінці, викликаючи `parser.js`.
    *   `nextPage()` — клікає на наступну сторінку.

#### 5. `src/utils/fileHandler.js`
Оскільки ви плануєте роботу з форматами, цей модуль має приймати масив об'єктів і перетворювати його.
*   **Методи:** `saveToCSV(data)`, `saveToJSON(data)`.

### Чому це сподобається клієнту на Upwork:

1.  **Scalability:** Ви можете легко додати нові функції (наприклад, фільтрацію за станом товару), просто додавши метод у клас.
2.  **Robustness:** Використання `stealth` плагіна та правильних затримок (`delay`) показує, що ви знаєте, як обходити захист.
3.  **Readability:** Код розбитий на малі функції (Single Responsibility Principle). Навіть нетехнічний клієнт зрозуміє структуру папок.

### Порада для реалізації:
Використовуйте `ES Modules` (`import/export`) замість `require`. Це сучасний стандарт Node.js, який краще виглядає в портфоліо 2026 року.
